Key Workflows
Flow 1 — Enrolled resident sees ad shelf
1
Resident navigates to Rent Credit Reporting page in Homebody
All feature flags enabled. Resident is RCR-enrolled. Property on allowlist.
2
Credit score ring and payment timeline load
Credit score 712 (+8 pts). 6-month on-time streak displayed.
3
Ad shelf loads asynchronously below credit content
WebFlow iFrame points to Growth Marketing CMS. 3 contextual offers visible.
4
Intersection Observer fires — offer_shelf_impression logged to Amplitude
surface=rcr, platform=homebody_rxp, enrollment_status=enrolled
5
Resident taps "Get a quote →" on Lemonade offer
offer_shelf_click fires. iFrame modal opens (simulates WebFlow CMS content).
Flow 2 — Unenrolled resident (no shelf)
1
Toggle rcr_enrolled flag to OFF in the Flags panel
2
RCR enrollment CTA appears. Ad shelf does not render.
Fail-closed. Zero footprint. Resident sees enrollment prompt instead.
Flow 3 — Kill switch activated
1
Toggle hb_rcr_ad_shelf_enabled to OFF in the Flags panel
2
Shelf collapses to dashed outline with fail-closed message
No code deploy. Immediate. Other surfaces (Payment Confirmation) unaffected.
Flow 4 — Homebody Your Way exclusion
1
Toggle homebody_your_way to ON (excluded)
2
Shelf does not render. Fail-closed message explains contractual exclusion.
Hard exclusion — enforced at platform level. Non-configurable.
Flow 5 — Shelf loading state
1
Shelf component mounts. offer_shelf_load_started fires.
2
Skeleton placeholder (3 gray cards) renders while eligibility + CMS fetch complete.
3
On success: skeleton replaced with offer cards. offer_shelf_impression fires when visible.
4
On failure: skeleton replaced with zero-footprint collapse. offer_shelf_load_failed fires.
Flow 6 — Empty CMS state (no offers)
1
Resident is eligible. Feature flags pass. CMS returns offers: [].
2
Shelf collapses to zero footprint. No error displayed to resident.
3
offer_shelf_load_failed fires with error_type: "cms_empty".
4
Growth Marketing notified via New Relic alert (zero-offer surface).

AI / Automation Opportunities
  • Automated offer relevance scoring: ML model scores which offer categories correlate with high RCR engagement (savings, credit tools, insurance) based on Amplitude impression + click signals.
  • Enrollment nudge optimization: For unenrolled residents, AI-driven messaging variant selection could improve enrollment CTR.
  • Anomaly detection: New Relic + ML to auto-flag shelf render degradation before manual thresholds trigger.
  • Complaint prediction: Model to identify properties likely to generate complaints before escalation thresholds are hit — enabling proactive client exclusion.

Error responses:

Pseudocode for eligibility:

function isShelfEligible(resident_id, property_id):
  enrollment = SELECT enrolled, status FROM homebody.rcr_enrollment 
               WHERE resident_id = ? AND property_id = ?
  
  IF enrollment IS NULL OR enrollment.enrolled = false OR enrollment.status != 'active':
    RETURN { eligible: false, reason: 'not_enrolled' }
  
  flags = evaluateFlags(surface='rcr', property_id=property_id)
  
  IF flags.hb_ad_shelf_global_kill = true:
    RETURN { eligible: false, reason: 'global_kill' }
  IF flags.hb_rcr_ad_shelf_enabled = false:
    RETURN { eligible: false, reason: 'surface_disabled' }
  IF property_id NOT IN flags.hb_rcr_ad_shelf_allowlist:
    RETURN { eligible: false, reason: 'property_not_allowlisted' }
  
  your_way = SELECT homebody_your_way FROM homebody.client_config WHERE property_id = ?
  IF your_way = true:
    RETURN { eligible: false, reason: 'your_way_exclusion' }
  
  RETURN { eligible: true }

Default behavior: If any check fails, returns error, or times out → eligible: false. Fail-closed.

Accessibility Requirements
  • Shelf container: role="complementary", aria-label="Featured financial offers"
  • Offer cards: role="article" within role="feed"
  • CTA buttons: Descriptive aria-label (e.g., "Open account with Marcus by Goldman Sachs")
  • Tab order: Offer cards are focusable in DOM order. Tab cycles through CTAs.
  • Focus management: When iframe modal opens, focus moves to modal. On close, focus returns to the CTA that triggered it.
  • Color contrast: All text meets WCAG 2.1 AA (4.5:1 for body, 3:1 for large text). Shelf label text (#94a3b8 on white) should be verified — may need darkening.
  • Screen reader: Offer count announced on shelf load ("3 financial offers available"). Kill switch states announced via aria-live="polite" region.
  • Motion: Respect prefers-reduced-motion for any animations.
